SUM Math
Adds all numbers in a range or list of values
▼

SUM adds together all numbers. You can provide individual cells, ranges, or a mix. One of Excel's most fundamental functions.

Syntax
=SUM(number1, [number2], ...)
Parameters
number1Required. First number, cell, or range to add.
number2…Optional. Up to 255 additional numbers/ranges.
Examples
FormulaDescriptionResult
=SUM(10, 20, 30)Direct values60
=SUM(A1:A10)Sum a rangeSum of A1–A10
=SUM(A1:A5, C1:C5)Multiple rangesBoth ranges combined
=SUM(A1:A10)-B1Sum minus a valueNet total
Keyboard Shortcut
Alt+=Auto-sum selected range
💡 Tip: Select a cell below/beside a column and press Alt+= to instantly insert =SUM() with the range auto-detected.

LET Math
Assign names to intermediate calculations within a formula
▼

LET assigns names to calculation results inside a formula. This makes complex formulas readable, avoids repeating calculations, and improves performance.

=LET(name1, value1, [name2, value2, ...], calculation)
FormulaDescription
=LET(tax, A1*0.18, total, A1+tax, total)Calculate tax then total — each used once
=LET(data, A1:A100, avg, AVERAGE(data), FILTER(data, data>avg))Filter values above average — data used twice, computed once
💡 Benefit: Without LET, a formula like =A1*0.18+A1 calculates A1 twice. LET computes intermediate values once and reuses them — cleaner and faster.

01
VLOOKUP Lookup
Searches a column and returns a value from another column
▼

VLOOKUP (Vertical Lookup) is one of the most used Excel functions. It searches down the first column of a table for a match, then returns a value from a specified column.

=VLOOKUP(lookup_value, table_array, col_index_num, [range_lookup])
lookup_valueThe value to search for (e.g., employee ID, product code)
table_arrayThe range containing the lookup table (lookup col must be FIRST)
col_index_numColumn number to return (1 = first column of table_array)
range_lookupFALSE = exact match (recommended); TRUE = approximate match
FormulaDescription
=VLOOKUP(A2, $E$2:$G$100, 2, FALSE)Exact match — find A2 in col E, return col F
=VLOOKUP(A2, Sheet2!$A:$C, 3, 0)Lookup across sheets

• #N/A — value not found (check exact spelling, use TRIM)
• #REF! — col_index exceeds table width
• Wrong values — forgot FALSE/0 for exact match
• Slow on large data — lookup column not locked with $
💡 Modern Excel: Replace VLOOKUP with XLOOKUP (Excel 2021+) — no column number needed, supports reverse lookup, and returns arrays: =XLOOKUP(A2, B:B, C:C, "Not Found")

Dynamic Arrays & Spill Behaviour 365 Core
How Excel 365 fundamentally changed formula results — arrays spill automatically
▼

Before Excel 365, a formula always returned exactly one value into one cell. Excel 365 introduced Dynamic Arrays — a single formula can now return multiple values that automatically spill into neighbouring cells. This single change makes dozens of previously complex tasks trivially simple.

The Spill Range Operator — #

When a formula spills, the entire output is called the spill range. Reference it using the # operator after the top-left cell.

Implicit Intersection & the @ Operator 365 Core
How 365 handles legacy formulas in a dynamic array world
▼

In older Excel, if you referenced an entire column (A:A) in a formula, Excel silently used implicit intersection — it automatically picked the value in the same row as the formula. In Excel 365, references to ranges would spill. The @ operator (Implicit Intersection Operator) restores the old single-value behaviour explicitly.

=@A:A — returns the value from column A in the SAME ROW (implicit intersection) =A:A — in 365, would spill the entire column =@SORT(A:A) — returns only the FIRST value from SORT's spill output =@UNIQUE(B:B) — returns only the first unique value
FormulaOld Excel BehaviourExcel 365 Behaviour
=A:AImplicit intersection (same row)Spills entire column — use @A:A to match old behaviour
=@A:ASame as =A:AExplicit: take value from same row only
=VLOOKUP(A:A, ...)Looked up same-row valueNow spills results for all rows — use @A:A inside if needed
📌 When you open old workbooks in 365: Excel automatically adds @ to formulas that previously relied on implicit intersection. You may see =@VLOOKUP(...) — this is Excel preserving old single-value behaviour. It still works correctly.

ISOMITTED 365 Only
Check if an optional LAMBDA parameter was not provided
▼

ISOMITTED is used exclusively inside LAMBDA functions to test whether an optional parameter was provided by the caller. Returns TRUE if the parameter was omitted, FALSE if it was supplied. Essential for building LAMBDA functions with default values.

=ISOMITTED(argument)

Native Checkboxes New 2024
True checkbox cells — insert, check/uncheck, use in formulas
▼

Excel 365 introduced native checkbox cells in 2024. Unlike the old Form Controls checkbox, these are directly inside cells and return TRUE/FALSE — no VBA or complex setup needed.

Inserting Checkboxes
Select cells→ Insert tab→ Checkbox
FormulaDescription

  • Options — Configure Excel behavior, add-ins, formulas, language

Conditional Formatting
▼

Apply formatting automatically based on cell values or formulas. Turns your data into a visual dashboard without charts.

  • Highlight Cell Rules — Greater than, less than, equal to, between, text containing, duplicate values
  • Top/Bottom Rules — Top 10 items, top 10%, bottom 10, above average, below average
  • Data Bars — Horizontal bar inside cells proportional to value (in-cell chart)
  • Color Scales — 2- or 3-color gradient across a range (heat map effect)
  • Icon Sets — Arrows, traffic lights, stars, check marks based on value thresholds
  • New Rule (Custom Formula) — Use formulas like =MOD(ROW(),2)=0 to stripe alternate rows
  • Manage Rules — View, edit, reorder, and delete existing rules
💡 Pro Tips:
• Use =$A1="Pending" (lock column, not row) to highlight entire rows
• Stack multiple rules — first TRUE rule wins (manage order carefully)
• Use ISBLANK(), ISNUMBER(), ISTEXT() in custom formulas

Table & Pivot Table
▼

Pivot Tables are Excel's most powerful data analysis tool. They allow you to summarize, group, and cross-tabulate large datasets in seconds.

  • Create Pivot Table — Insert → Pivot Table; choose source data and placement
  • Field Areas — Drag fields to: Filters (report filter), Columns (cross-tab), Rows (groups), Values (aggregate)
  • Value Settings — Right-click → Value Field Settings: Sum, Count, Average, Max, Min, % of total, etc.
  • Group — Group dates by month/quarter/year; group numbers into bins
  • Calculated Field — Create custom formulas within the Pivot Table (e.g., Profit = Revenue - Cost)
  • Refresh — Alt+F5 or right-click → Refresh after source data changes
  • Pivot Chart — Insert a chart linked to a Pivot Table; updates with filters automatically
  • Slicers — Visual filter buttons; click to filter Pivot Table without using dropdown arrows
Alt+F5Refresh Pivot Ctrl+TCreate Table
💡 Best Practice: Convert source data to a Table (Ctrl+T) before creating a Pivot — the Pivot will auto-include new rows when you refresh!
